================================================================================
  LSP BRIDGE MODULE — Dual-Mode Language Server Client (V1 Custom + V2 Built-in)
================================================================================

PROJECT: Monaco-Vanced LSP Bridge
MODULE: plugins/language/lsp-bridge-module/

GOAL:
Provide TWO LSP client implementations that connect Monaco to any
LSP-compatible language server over WebSocket:

  Version 1 (Custom)  — Direct WebSocket + manual JSON-RPC 2.0 framing.
                         Lightweight, full control over retry/reconnect.
  Version 2 (Built-in) — Full LSP lifecycle using vscode-ws-jsonrpc for
                         transport and the standard LSP initialize handshake.
                         Registers 24 Monaco providers via LSP↔Monaco converters.
                         Handles textDocument/didOpen, didChange, diagnostics.

Both clients are registered. User chooses which to use via settings.
Each mode establishes its own WebSocket, manages its own lifecycle, and can
be switched at runtime without reloading.

This module does NOT own language data, CDN fetching, or provider data —
those belong to context-engine. This module manages WebSocket transport,
LSP protocol, provider registration, and exposes clients for other modules.

DEFAULT SERVER:
  URL:      Configurable via settings (default from env LSP_URL)
  Default:  https://monaco-lsp-hub.onrender.com/
  Protocol: WebSocket + JSON-RPC 2.0

REFERENCE:
  Architecture patterns adapted from terminus-web/lib/lsp/ (own project).
  Adapted to MonacoPlugin architecture with ctx.provide() pattern.

================================================================================

  The module supports two client versions. Both can run simultaneously or
  the user can choose one. Selection is via settings: lsp.mode

  ┌────────────────────┬───────────────────────────────────────────────────┐
  │ Aspect             │ V1 (Custom)           │ V2 (Built-in)            │
  ├────────────────────┼───────────────────────┼──────────────────────────┤
  │ Transport          │ Raw WebSocket         │ vscode-ws-jsonrpc        │
  │ Framing            │ Manual JSON.stringify  │ MessageReader/Writer     │
  │ Handshake          │ None — fire requests  │ Full initialize/         │
  │                    │ directly              │ initialized handshake    │
  │ Provider reg       │ Generic bridge        │ 24 typed Monaco          │
  │                    │ (ProviderFactory)     │ providers via converters │
  │ didOpen/didChange  │ Not managed           │ Automatic — tracks open  │
  │                    │ (consumer handles)    │ models + content changes │
  │ Diagnostics        │ Via notification sub  │ Automatic markers bridge │
  │ Server caps        │ Not tracked           │ Parsed from initialize   │
  │                    │                       │ response, guards features│
  │ Reconnect          │ Same retry logic      │ Same retry logic         │
  │ Best for           │ Lightweight, custom   │ Full LSP experience      │
  │                    │ protocol extensions   │ with all 24 providers    │
  └────────────────────┴───────────────────────┴──────────────────────────┘

──────────────────────────────────────────────────────────────
4A — CONNECTION LIFECYCLE
──────────────────────────────────────────────────────────────

  class LspConnectionManager {

    private state: LspConnectionState = "disconnected";
    private retryCount: number = 0;
    private retryTimer: ReturnType<typeof setTimeout> | null = null;
    private hasConnectedOnce: boolean = false;

    constructor(
      private eventBus: EventBus,
      private config: LspConnectionConfig,
    ) {}

    // ── State machine ─────────────────────────────────────

    transition(newState: LspConnectionState): void
      → Updates state
      → Emits corresponding LspEvent
      → Updates status bar text

    getState(): LspConnectionState
    isConnected(): boolean

    // ── Retry logic ───────────────────────────────────────

    scheduleRetry(connectFn: () => Promise<void>): void
      → retryCount++
      → If retryCount >= maxRetries → transition("failed"), fire failure notification
      → Else → wait retryIntervalMs, call connectFn()

    resetRetries(): void
      → retryCount = 0

    cancelRetry(): void
      → Clear retryTimer

    // ── First-connect tracking ────────────────────────────

    markFirstConnect(): boolean
      → If !hasConnectedOnce → set true, return true (fire notification)
      → Else return false (skip notification)
  }

──────────────────────────────────────────────────────────────
4B — RETRY BEHAVIOR
──────────────────────────────────────────────────────────────

  ┌──────────┬──────────────────────────────────────────────────┐
  │ Attempt  │ Timing                                           │
  ├──────────┼──────────────────────────────────────────────────┤
  │ 1        │ Immediate (on first connect or manual retry)     │
  │ 2–10     │ Every 15 seconds after previous failure          │
  │ After 10 │ STOP. Show error notification. Wait for manual.  │
  └──────────┴──────────────────────────────────────────────────┘

SECTION 5 — VERSION 1: CUSTOM JSON-RPC CLIENT
================================================================================

  Lives in: plugins/language/lsp-bridge-module/v1-custom-client.ts

  Direct WebSocket + manual JSON-RPC 2.0 framing. No LSP initialize
  handshake — sends RPC requests immediately on connect. Lightweight,
  suitable for custom protocol extensions beyond standard LSP.

──────────────────────────────────────────────────────────────
5A — CUSTOM CLIENT CLASS
──────────────────────────────────────────────────────────────

  class CustomLspClient {

    private ws: WebSocket | null = null;
    private manager: LspConnectionManager;
    private pendingRequests: Map<number, PendingRequest>;
    private notificationHandlers: Map<string, Set<(params: unknown) => void>>;
    private nextRequestId: number = 1;

    constructor(
      private eventBus: EventBus,
      private config: LspConnectionConfig,
    ) {
      this.manager = new LspConnectionManager(eventBus, config);
    }

    // ── Connect ───────────────────────────────────────────

    async connect(): Promise<void>
      → manager.transition("connecting")
      → Opens new WebSocket(config.url) directly
      → On open:
          manager.transition("connected")
          manager.markFirstConnect() → notification if first time
      → On message:
          Parse JSON → route to pending request or notification handler
      → On error/close:
          manager.scheduleRetry(() => this.connect())

    // ── Disconnect ────────────────────────────────────────

    disconnect(): void
      → manager.cancelRetry()
      → Close WebSocket
      → Reject all pending requests with DisconnectedError
      → manager.transition("disconnected")

    // ── RPC Methods ───────────────────────────────────────

    async request<T>(method: string, params?: unknown): Promise<T>
      → Generates auto-increment request id
      → Sends { jsonrpc: "2.0", id, method, params }
      → Returns promise that resolves on matching response id
      → Emits LspEvent.RequestSent
      → On response: emits LspEvent.ResponseReceived
      → Timeout: config.timeoutMs → rejects with TimeoutError, emits LspEvent.RequestTimeout
      → If disconnected: rejects immediately

    notify(method: string, params?: unknown): void
      → Sends { jsonrpc: "2.0", method, params } (no id, no response)

    onNotification(method: string, handler: (params: unknown) => void): IDisposable
      → Registers handler for server-sent notifications
      → Returns disposable to unregister

    // ── State ─────────────────────────────────────────────

    getState(): LspConnectionState → delegates to manager
    isConnected(): boolean → delegates to manager
    retry(): void → manager.resetRetries() + connect()
  }

SECTION 6 — VERSION 2: BUILT-IN MONACO LSP CLIENT
================================================================================

  Lives in: plugins/language/lsp-bridge-module/v2-builtin-client.ts

  Full LSP lifecycle. Opens WebSocket, performs initialize/initialized
  handshake, parses server capabilities, registers 24 typed Monaco
  providers, tracks open documents, bridges diagnostics to markers.

  Uses converters.ts (LSP ↔ Monaco type conversions).

──────────────────────────────────────────────────────────────
6A — BUILTIN CLIENT CLASS
──────────────────────────────────────────────────────────────

  class BuiltinLspClient {

    private ws: WebSocket | null = null;
    private manager: LspConnectionManager;
    private pendingRequests: Map<number, PendingRequest>;
    private nextRequestId: number = 1;
    private serverCapabilities: Record<string, any> = {};
    private providerRegistration: LspProviderRegistration | null = null;
    private documentVersions: Map<string, number> = new Map();

    constructor(
      private eventBus: EventBus,
      private config: LspConnectionConfig,
      private monaco: typeof import("monaco-editor"),
      private editor: monacoNs.editor.ICodeEditor | null,
    ) {
      this.manager = new LspConnectionManager(eventBus, config);
    }

    // ── Connect ───────────────────────────────────────────

    async connect(languageId: string): Promise<LspConnection>
      → manager.transition("connecting")
      → Builds wsUrl via buildLSPWebSocketUrl(config.url, languageId)
      → If null → throw "Unsupported language: ${languageId}"
      → Opens new WebSocket(wsUrl)
      → On open:
          await performInitializeHandshake(languageId)
          manager.transition("connected")
          markFirstConnect → notification if first
          registerAllProviders(languageId)
          sendDidOpen(languageId) for current model
          subscribe to model content changes → sendDidChange
      → On message:
          Parse JSON → route to pending request or notification
          Handle "textDocument/publishDiagnostics" → set Monaco markers
          Handle "window/showMessage" → forward to notification service
          Handle "window/logMessage" → console log (errors/warnings → notification)
      → On error/close:
          dispose providers
          manager.scheduleRetry(() => this.connect(languageId))
      → Returns LspConnection { client, providers, dispose, isConnected }

──────────────────────────────────────────────────────────────
6B — LSP INITIALIZE HANDSHAKE
──────────────────────────────────────────────────────────────

  private async performInitializeHandshake(languageId: string): Promise<void>

    1. Send "initialize" request:
       {
         jsonrpc: "2.0",
         id: 1,
         method: "initialize",
         params: {
           processId: null,
           rootUri: config.rootUri ?? "file:///workspace",
           capabilities: {
             textDocument: {
               completion: { completionItem: { snippetSupport: true } },
               hover: { contentFormat: ["markdown", "plaintext"] },
               signatureHelp: { signatureInformation: { parameterInformation: { labelOffsetSupport: true } } },
               synchronization: { didSave: true, willSave: false },
               codeAction: { codeActionLiteralSupport: { codeActionKind: { valueSet: [] } } },
               semanticTokens: { requests: { full: true, range: true }, tokenTypes: [...], tokenModifiers: [...] },
               inlayHint: { resolveSupport: { properties: ["tooltip", "textEdits", "label.tooltip", "label.location", "label.command"] } },
             },
             workspace: {
               workspaceFolders: true,
               didChangeConfiguration: { dynamicRegistration: false },
             },
           },
           initializationOptions: {},
         },
       }

    2. Receive "initialize" response:
       → Parse result.capabilities → store as this.serverCapabilities
       → Used to guard which providers to register
         (e.g. only register completionProvider if capabilities.completionProvider exists)

    3. Send "initialized" notification:
       { jsonrpc: "2.0", method: "initialized", params: {} }

──────────────────────────────────────────────────────────────
6C — DOCUMENT SYNCHRONIZATION
──────────────────────────────────────────────────────────────

  private sendDidOpen(languageId: string): void
    → Gets current editor model
    → Sends notification: "textDocument/didOpen" with:
      { textDocument: { uri: model.uri, languageId, version: 1, text: model.getValue() } }
    → Records version in documentVersions map

  private setupDidChange(): IDisposable
    → Subscribes to model.onDidChangeContent
    → On each change:
        version++ in documentVersions
        Sends notification: "textDocument/didChange" with:
        { textDocument: { uri, version }, contentChanges: [{ text: model.getValue() }] }
    → Full document sync (not incremental) for simplicity

──────────────────────────────────────────────────────────────
6D — DIAGNOSTICS BRIDGE
──────────────────────────────────────────────────────────────

  Listens for server notification: "textDocument/publishDiagnostics"

  On receive:
    → params: { uri: string, diagnostics: Diagnostic[] }
    → Convert diagnostics via toMonacoMarkers(monaco, diagnostics)
    → Apply: monaco.editor.setModelMarkers(model, `lsp-${languageId}`, markers)

  Only applies if the model URI matches the diagnostics URI.

──────────────────────────────────────────────────────────────
6E — SERVER MESSAGE HANDLING
──────────────────────────────────────────────────────────────

  Listens for server notifications:
    "window/showMessage"  → forward to notificationService as toast
    "window/logMessage"   → console log, only surface errors/warnings as notifications

  LSPMessageType mapping:
    1 (Error)   → "error"
    2 (Warning) → "warning"
    3 (Info)    → "info"
    4 (Log)     → "log"
    5 (Debug)   → "debug"

SECTION 7 — LSP ↔ MONACO CONVERTERS
================================================================================

  Lives in: plugins/language/lsp-bridge-module/converters.ts

  Bidirectional converters between LSP protocol types and Monaco editor
  types. Used exclusively by V2 (Built-in) client for provider registration.

  Adapter pattern — translates between two incompatible interfaces
  (LSP ↔ Monaco) without coupling either side.

──────────────────────────────────────────────────────────────
7A — POSITION / RANGE / LOCATION
──────────────────────────────────────────────────────────────

  toMonacoPosition(pos: lsp.Position): IPosition
    → { lineNumber: pos.line + 1, column: pos.character + 1 }

  fromMonacoPosition(pos: IPosition): lsp.Position
    → Position.create(pos.lineNumber - 1, pos.column - 1)

  toMonacoRange(range: lsp.Range): IRange
    → 0-based → 1-based conversion for all four coordinates

  fromMonacoRange(range: IRange): lsp.Range
    → 1-based → 0-based conversion for all four coordinates

──────────────────────────────────────────────────────────────
7B — COMPLETION
──────────────────────────────────────────────────────────────

  Lazy-init map: LSP_TO_MONACO_COMPLETION_KIND (25 entries)
    lsp.CompletionItemKind.Text       → monaco.languages.CompletionItemKind.Text
    lsp.CompletionItemKind.Method     → ...Method
    lsp.CompletionItemKind.Function   → ...Function
    ... (all 25 LSP completion kinds mapped 1:1)

  toMonacoCompletionItem(monaco, item: lsp.CompletionItem, range): CompletionItem
    → Maps: label, kind, detail, documentation (string or markdown),
      insertText, insertTextRules (snippet detection), range, sortText,
      filterText, preselect, tags

──────────────────────────────────────────────────────────────
7C — HOVER
──────────────────────────────────────────────────────────────

  toMonacoHover(hover: lsp.Hover): monacoNs.languages.Hover
    → Handles: string, string[], MarkedString, MarkupContent
    → Wraps code blocks with language fences for MarkedString objects
    → Returns { contents: IMarkdownString[], range? }

──────────────────────────────────────────────────────────────
7D — DIAGNOSTICS
──────────────────────────────────────────────────────────────

  toMonacoSeverity(monaco, severity): MarkerSeverity
    → Error→Error, Warning→Warning, Information→Info, Hint→Hint

  toMonacoMarkers(monaco, diagnostics): IMarkerData[]
    → Maps each diagnostic: severity, message, source, code, range coordinates

──────────────────────────────────────────────────────────────
7E — SIGNATURE HELP
──────────────────────────────────────────────────────────────

  toMonacoSignatureHelp(sh: lsp.SignatureHelp): SignatureHelpResult
    → Maps signatures with label, documentation, parameters
    → Includes activeSignature, activeParameter
    → Returns { value: {...}, dispose: () => {} }

──────────────────────────────────────────────────────────────
7F — DEFINITION / REFERENCES

SECTION 13 — MODULE INIT / PLUGIN CONTRACT
================================================================================

  Lives in: plugins/language/lsp-bridge-module/index.ts

  export const LspBridgePlugin: MonacoPlugin = {
    id: "lsp-bridge",
    name: "LSP Bridge",
    version: "2.0.0",

    async init(ctx: ModuleContext) {
      const config = ctx.settings.get("lsp");

      if (!config.enabled) return;

      // ── Create clients based on mode ──────────────────

      let customClient: CustomLspClient | null = null;
      let builtinClient: BuiltinLspClient | null = null;

      const mode: LspMode = config.mode ?? "builtin";

      if (mode === "custom" || mode === "both") {
        customClient = new CustomLspClient(ctx.eventBus, config);
      }
      if (mode === "builtin" || mode === "both") {
        builtinClient = new BuiltinLspClient(ctx.eventBus, config, ctx.monaco, ctx.editor);
      }

      // ── Expose for other modules ──────────────────────

      ctx.provide("lsp.mode", mode);
      ctx.provide("lsp.customClient", customClient);
      ctx.provide("lsp.builtinClient", builtinClient);
      ctx.provide("lsp.hasLSPSupport", hasLSPSupport);
      ctx.provide("lsp.buildUrl", (langId: string) => buildLSPWebSocketUrl(config.url, langId));

      // For V1: also expose the legacy interface names
      if (customClient) {
        ctx.provide("lsp.rpcClient", customClient);
        ctx.provide("lsp.connection", customClient);
        ctx.provide("lsp.bridge", new LspProviderBridge(customClient, ctx.monaco));
      }

      // ── Register status bar ───────────────────────────

      ctx.statusBar.register({
        id: "lsp-status",
        alignment: "right",
        priority: 100,
        text: "$(circle-slash) LSP: Disconnected",
        tooltip: `LSP: ${config.url} (${mode})`,
        onClick: () => handleStatusBarClick(customClient, builtinClient, mode, ctx),
      });

      // ── Listen for settings changes ───────────────────

      ctx.settings.onChange("lsp.url", (newUrl) => {
        customClient?.disconnect();
        builtinClient?.disconnect();
        config.url = newUrl;
        // Reconnect happens via autoConnect logic
        if (config.autoConnect) {
          customClient?.connect();
          // builtinClient reconnects per-language via connectForLanguage()
        }
        ctx.notifications.info("LSP URL changed — reconnecting...", {
          category: "lsp",
          source: "Language Server",
        });
      });

      ctx.settings.onChange("lsp.mode", (newMode: LspMode) => {
        // Dispose old clients
        customClient?.disconnect();
        builtinClient?.disconnect();
        customClient = null;
        builtinClient = null;

        // Create new clients
        if (newMode === "custom" || newMode === "both") {
          customClient = new CustomLspClient(ctx.eventBus, config);
        }
        if (newMode === "builtin" || newMode === "both") {
          builtinClient = new BuiltinLspClient(ctx.eventBus, config, ctx.monaco, ctx.editor);
        }

        // Update provides
        ctx.provide("lsp.mode", newMode);
        ctx.provide("lsp.customClient", customClient);
        ctx.provide("lsp.builtinClient", builtinClient);

        ctx.eventBus.emit(LspEvent.ModeChanged, { mode: newMode });
        ctx.notifications.info(`LSP mode changed to ${newMode}`, {
          category: "lsp",
          source: "Language Server",
        });

        // Auto-connect new clients
        if (config.autoConnect) {
          customClient?.connect();
        }
      });

      // ── Auto-connect on startup ───────────────────────

      if (config.autoConnect) {
        customClient?.connect();
        // V2 builtin client connects per-language when editor opens a file
        // → see connectForLanguage() triggered by editor.onDidChangeModel
      }

      // ── V2: Auto-connect on language change ───────────

      if (builtinClient) {
        const connectForLanguage = (model: monacoNs.editor.ITextModel) => {
          const langId = model.getLanguageId();
          if (hasLSPSupport(langId)) {
            builtinClient!.connect(langId);
          }
        };

        // Connect for current model
        const currentModel = ctx.editor?.getModel();
        if (currentModel) connectForLanguage(currentModel);

        // Connect when model changes
        ctx.editor?.onDidChangeModel((e) => {
          if (e.newModelUrl) {
            const model = ctx.monaco.editor.getModel(e.newModelUrl);
            if (model) connectForLanguage(model);
          }
        });
      }

      // ── Cleanup ───────────────────────────────────────

      ctx.onDestroy(() => {
        customClient?.disconnect();
        builtinClient?.disconnect();
      });
    },

    async destroy() {
      // Connection cleanup handled by onDestroy
    },
  };

SECTION 15 — SERVER LIVENESS POLLING (PING/PONG)
================================================================================

  The LSP module includes a keep-alive health check that periodically pings
  the LSP server to verify liveness. If the server stops responding, the
  module transitions to reconnecting state automatically.

──────────────────────────────────────────────────────────────
15A — PING ENDPOINT
──────────────────────────────────────────────────────────────

  HTTP GET  ${lsp.url}/ping
  Expected: 200 OK  body: "pong"

  The ping endpoint is a lightweight HTTP GET (not WebSocket) that verifies
  the server process is alive. It's separate from the WebSocket connection
  to detect server restarts even when the WS connection appears open.

──────────────────────────────────────────────────────────────
15B — POLLING BEHAVIOR
──────────────────────────────────────────────────────────────

  - Enabled by default (lsp.ping.enabled = true)
  - Interval: lsp.ping.intervalMs (default: 15000 — 15 seconds)
  - Minimum interval: 15 seconds (15000ms — enforced, ignores lower values)
  - Maximum interval: 2 minutes (120000ms — enforced, clamps higher values)
  - Only polls when connection state is "connected"
  - Stops polling on disconnect/reconnecting/failed
  - Resumes polling on successful reconnect
  - Timeout per ping: 5000ms (5 seconds) — if no response, treat as failure

  FAILURE HANDLING:
  - 1 missed ping → emit LspEvent.PingTimeout, log warning
  - 2 consecutive missed pings → transition to "reconnecting", start retry
  - On successful ping after failure → reset missed count, emit LspEvent.PingSuccess

  ┌──────────────────────┬─────────────────────────────────────────────────┐
  │ Ping result          │ Action                                          │
  ├──────────────────────┼─────────────────────────────────────────────────┤
  │ 200 + "pong"         │ Reset missed count, emit PingSuccess            │
  │ Non-200 / timeout    │ Increment missed count                          │
  │ 2+ consecutive miss  │ Transition to "reconnecting", trigger retry     │
  │ Network error        │ Count as missed ping                            │
  └──────────────────────┴─────────────────────────────────────────────────┘

──────────────────────────────────────────────────────────────
15C — LIVENESS EVENTS
──────────────────────────────────────────────────────────────

  LspEvent.PingSuccess    = "lsp:ping-success"     { latencyMs }
  LspEvent.PingTimeout    = "lsp:ping-timeout"     { missedCount }
  LspEvent.PingFailed     = "lsp:ping-failed"      { missedCount, error }

──────────────────────────────────────────────────────────────
15D — SETTINGS
──────────────────────────────────────────────────────────────

  {
    "lsp.ping.enabled": {
      "type": "boolean",
      "default": true,
      "description": "Enable periodic server liveness ping"
    },
    "lsp.ping.intervalMs": {
      "type": "number",
      "default": 15000,
      "minimum": 15000,
      "maximum": 120000,
      "description": "Ping interval in milliseconds (min: 15s, max: 2min)"
    }
  }

  LIVE SETTINGS UPDATE:
  - Changing lsp.ping.enabled to false → stop polling immediately
  - Changing lsp.ping.intervalMs → applies to NEXT polling cycle
  - Clamped at runtime: Math.max(15000, Math.min(120000, value))


================================================================================
SECTION 16 — WHAT TO AVOID
================================================================================

  - ❌ Don't reconnect infinitely (max 10 retries, then stop)
  - ❌ Don't spam notifications on every reconnect (only first connect + final failure)
  - ❌ Don't block editor startup waiting for LSP (it's async, editor works without it)
  - ❌ Don't hardcode the LSP URL (always from settings or env)
  - ❌ Don't retry faster than 5 seconds (server rate limiting)
  - ❌ Don't cache WebSocket instance on reconnect (create new one each time)
  - ❌ Don't forget to clean up pending requests on disconnect
  - ❌ Don't own provider data or CDN logic (that's context-engine's responsibility)
  - ❌ Don't register V2 providers without checking server capabilities first
  - ❌ Don't run V1 and V2 on the same WebSocket (each gets its own connection)
  - ❌ Don't send didOpen/didChange in V1 mode (that's V2's responsibility)
  - ❌ Don't mix converter imports in V1 code (converters are V2-only)
  - ❌ Don't forget to dispose providers when switching modes or disconnecting
